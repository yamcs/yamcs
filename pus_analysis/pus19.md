# PUS ST[19] Event-Action Service — Analysis & Implementation Plan

**Spec reference**: ECSS-E-ST-70-41C §6.19 (requirements) and §8.19 (packet definitions)
**Required subtypes**: TC[19,1], TC[19,2], TC[19,4], TC[19,5], TC[19,6], TM[19,7], TC[19,8], TC[19,9], TC[19,10], TM[19,11]

---

## a) General Context

PUS ST[19] is the **Event-Action Service**. It provides the capability to define on-board actions that
are autonomously executed when specific on-board events occur. The subservice reacts to events
generated by associated event reporting subservices (ST[05]) by releasing the request stored for
each matching event-action definition.

### Key characteristics

| Property | Value |
|----------|-------|
| PUS service type | 19 |
| Sub-service | Event-action subservice |
| Coupled with | ST[05] event reporting |
| State maintained | Global function status + per-definition status |
| Background tasks | Event listener; action trigger thread |
| New pus_dt.xml types needed | None (use existing uint8/uint16/uint32) |
| Major XTCE limitation | Embedded TC packet in TC[19,1]/TM[19,11] not XTCE-expressible |

### Spec-defined message types (§8.19.2)

| Subtype | Direction | Name | In scope |
|---------|-----------|------|----------|
| 1 | TC | Add event-action definitions | YES |
| 2 | TC | Delete event-action definitions | YES |
| 3 | TC | Delete all event-action definitions | no |
| 4 | TC | Enable event-action definitions | YES |
| 5 | TC | Disable event-action definitions | YES |
| 6 | TC | Report status of each event-action definition | YES |
| 7 | TM | Event-action status report | YES |
| 8 | TC | Enable the event-action function | YES |
| 9 | TC | Disable the event-action function | YES |
| 10 | TC | Report event-action definitions | YES |
| 11 | TM | Event-action definition report | YES |

### Two-level state model

```
event-action function status  (global: enabled/disabled)
│
└─ if ENABLED → react to events
   │
   └─ for each event-action definition:
         event-action status  (per-definition: enabled/disabled)
         │
         └─ if ENABLED + matching event fires → release request TC
```

Key rules (§6.19.5):
- If function is disabled → no action triggered for any definition
- Disabling function does NOT change individual definition statuses
- Disabling a definition does NOT reset the function status

### Event-action definition structure (§6.19.4b)

Each definition contains:
1. **Event system identifier**: `(application_process_id, event_definition_id)` — identifies which ST[05] event triggers this action
2. **Request**: the TC packet to release when the event fires

In simplified PUS (test_yamcs style): `application_process_id` and `event_definition_id` are both encoded as `uint16`. The request is a raw TC packet (variable length).

### Enumeration values (§8.19.3, Tables 8-19/8-20)

| Field | Raw 0 | Raw 1 |
|-------|-------|-------|
| event-action function status | "disabled" | "enabled" |
| event-action status | "disabled" | "enabled" |

## b) Per-subtype Implementation Plan

> **Simplified PUS convention** (test_yamcs style): 6B CCSDS + 1B service_type + 1B service_subtype.
> No full PUS secondary header. APID=190 (0xBE) for all ST[19] packets.

### TC[19,1] — Add event-action definitions

**Spec §8.19.2.1 / Figure 8-212:**

```
N (uint8)
repeated N times:
  application_process_id  (uint16 / enumerated)
  event_definition_id     (uint16 / enumerated)
  request                 (TC packet — variable length, spec: "TC packet")
```

**Packet layout:**

```
Byte  0-5 : CCSDS primary header (TC, sec_hdr=1, APID=190)
Byte  6   : service_type = 19
Byte  7   : service_subtype = 1
Byte  8   : N (uint8) — number of definitions to add
Byte  9+  : N × { app_process_id:uint16, event_def_id:uint16, request_tc_len:uint16, request_tc_bytes:uint8[...] }
```

**NOTE on embedded TC packet**: The spec says the request field is a raw TC packet with no
length prefix. For the simplified simulator, we add a non-standard `request_tc_len:uint16`
prefix so the packet boundary can be determined. This is the pragmatic workaround for a
fundamental XTCE limitation (see §c gaps).

**XTCE encoding (pus19.xml):**

```xml
<!-- Request entry aggregate (simplified: length-prefixed raw bytes) -->
<AggregateArgumentType name="EventActionEntryType">
    <MemberList>
        <Member typeRef="/dt/uint16" name="app_process_id" />
        <Member typeRef="/dt/uint16" name="event_def_id" />
        <Member typeRef="/dt/uint16" name="request_tc_len" />
        <!-- request_tc_bytes: variable-length raw bytes — XTCE cannot express;
             handle in simulator Python code only -->
    </MemberList>
</AggregateArgumentType>
```

The `request_tc_bytes` field cannot be expressed as a dynamic-size XTCE argument. Only the
fixed header portion is declared in XTCE; Python parses remaining bytes manually.

**Python simulator code:**

```python
def parse_tc_19_1(data: bytes) -> list:
    """Parse TC[19,1] — returns list of (app_pid, event_id, request_bytes)."""
    offset = 8  # after CCSDS + svc/sub
    n = data[offset]; offset += 1
    entries = []
    for _ in range(n):
        app_pid = struct.unpack_from(">H", data, offset)[0]; offset += 2
        event_id = struct.unpack_from(">H", data, offset)[0]; offset += 2
        req_len = struct.unpack_from(">H", data, offset)[0]; offset += 2
        req_bytes = data[offset:offset + req_len]; offset += req_len
        entries.append((app_pid, event_id, req_bytes))
    return entries
```

**Simulator state update**: for each entry, store in `ea_definitions[(app_pid, event_id)] = {'request': req_bytes, 'status': 'disabled'}` (new) or replace request (existing).

### TC[19,4] — Enable event-action definitions

**Spec §8.19.2.4 / Figure 8-214:**

Same structure as TC[19,2]: `N + N×{app_process_id, event_def_id}`.

**N=0 semantics**: `TC[19,4]` with N=0 is not a valid "enable all" convention. Instead, per §6.19.7.1b, a request to enable all definitions contains **exactly one instruction to enable all**, with no argument (zero-payload instruction). In XTCE this means we use **two MetaCommands** (same pattern as ST[15]):

```xml
<!-- Specific definitions -->
<MetaCommand name="TC_19_4_specific" shortDescription="TC[19,4] Enable specific event-action definitions">
    <ArgumentList>
        <Argument argumentTypeRef="/dt/uint8" name="num_entries" />
        <Argument argumentTypeRef="EventIdArrayType" name="event_ids" />
    </ArgumentList>
    <CommandContainer name="TC_19_4_specific">
        <EntryList>
            <!-- CCSDS header with APID=190, service=19, subtype=4 -->
            ...
            <ArgumentRefEntry argumentRef="num_entries" />
            <ArgumentRefEntry argumentRef="event_ids" />
        </EntryList>
    </CommandContainer>
</MetaCommand>

<!-- Enable ALL definitions: N omitted entirely (zero-payload) -->
<MetaCommand name="TC_19_4_all" shortDescription="TC[19,4] Enable ALL event-action definitions">
    <!-- No ArgumentList -->
    <CommandContainer name="TC_19_4_all">
        <EntryList>
            <!-- CCSDS header + service=19, subtype=4 only -->
        </EntryList>
    </CommandContainer>
</MetaCommand>
```

Python disambiguates at runtime: if `len(data) == 8` (no payload after subtype byte) → enable all; else parse N + array.

### TM[19,7] — Event-action status report

**Spec §8.19.2.7 / Figure 8-216:**

```
N (uint8)
repeated N times:
  application_process_id  (uint16 / enumerated)
  event_definition_id     (uint16 / enumerated)
  event_action_status     (uint8: 0=disabled, 1=enabled)
```

**Packet layout:**

```
Byte  0-5 : CCSDS (TM, APID=190)
Byte  6   : service_type = 19
Byte  7   : service_subtype = 7
Byte  8   : N (uint8)
Byte  9+  : N × { app_process_id:uint16, event_def_id:uint16, ea_status:uint8 }  (5 bytes each)
```

                    <LinearAdjustment intercept="-1" />
                </DynamicValue>

    </DimensionList>
</ArrayParameterType>

<SequenceContainer name="TM_19_7" shortDescription="TM[19,7] Event-action status report">

        <ParameterRefEntry parameterRef="num_entries" />

**Python builder:**

```python
def build_tm_19_7(tm_sock: socket.socket) -> None:
    entries = list(ea_definitions.items())
    n = len(entries)
    payload = struct.pack(">B", n)
    for (app_pid, event_id), defn in entries:
        status = 1 if defn['status'] == 'enabled' else 0
        payload += struct.pack(">HHB", app_pid, event_id, status)
    _send_tm(tm_sock, subtype=7, data=payload)
```

### TC[19,10] — Report event-action definitions

**Spec §8.19.2.10 / Figure 8-217:**

```
N (uint8)
repeated N times:
  application_process_id  (uint16)
  event_definition_id     (uint16)
```

**N=0 semantics**: "To report all event-action definitions, N shall be set to 0" (§8.19.2.10c). This is **in-band encoding** (N=0 in payload), unlike TC[19,4]/[19,5]. The spec uses N=0 as the "report all" sentinel directly.

**XTCE**: Two MetaCommands:
- `TC_19_10_specific` — N>0 + array of event IDs
- `TC_19_10_all` — zero payload (N omitted; simulator interprets as "all")

Alternatively: single MetaCommand with `num_entries:uint8` argument; Python checks if `num_entries==0` at runtime.

**Python response**: Generates `TM[19,11]` with the requested definitions (or all if N=0).

### TM[19,11] — Event-action definition report

**Spec §8.19.2.11 / Figure 8-218:**

```
N (uint8)
repeated N times:
  application_process_id  (uint16 / enumerated)
  event_definition_id     (uint16 / enumerated)
  event_action_status     (uint8: enumerated)
  request                 (TC packet — variable length)
```

**Same embedded TC problem as TC[19,1]**: the request field is a full TC packet. For the
simplified simulator, we prefix with `request_tc_len:uint16`.

**Packet layout (simplified, with length prefix):**

```
Byte  0-5 : CCSDS (TM, APID=190)
Byte  6   : service_type = 19
Byte  7   : service_subtype = 11
Byte  8   : N (uint8)
Byte  9+  : N × { app_pid:uint16, event_id:uint16, ea_status:uint8,
                  req_len:uint16, req_bytes:uint8[req_len] }
```

**XTCE**: Only the fixed portion per entry is expressible. The `req_bytes` variable-length blob
is not. YAMCS will decode `app_pid`, `event_id`, and `ea_status` per entry; `req_bytes` is raw.

**Python builder:**

```python
def build_tm_19_11(definitions_to_report: list) -> bytes:
    n = len(definitions_to_report)
    payload = struct.pack(">B", n)
    for (app_pid, event_id) in definitions_to_report:
        defn = ea_definitions[(app_pid, event_id)]
        status = 1 if defn['status'] == 'enabled' else 0
        req = defn['request']
        payload += struct.pack(">HHB", app_pid, event_id, status)
        payload += struct.pack(">H", len(req)) + req
    return payload
```

## c) Gaps / Shortcomings

| # | Gap | Severity | Impact | Fix / Workaround |
|---|-----|----------|--------|-----------------|
| 1 | **Embedded TC packet in TC[19,1] and TM[19,11]** | **High** | Not XTCE-expressible; YAMCS cannot encode/decode the request field automatically | Add non-standard `request_tc_len:uint16` length prefix; handle in Python manually. YAMCS TC editor cannot build TC[19,1] without custom scripting. |
| 2 | **N=0 "all definitions" in TC[19,4]/TC[19,5]** | Medium | XTCE does not model conditional semantics (N=0 = all vs N>0 = specific) | Use two MetaCommands per subtype: `_specific` (with args) and `_all` (zero payload). Python disambiguates at runtime. |
| 3 | **Enumerated types for app_process_id and event_def_id** | Minor | Spec says "enumerated"; mission-specific values not known at design time | Use `uint16` as raw integer; add YAMCS enumeration mappings post-hoc if needed. |
| 4 | **No PUS-1 ACK/NACK in test_yamcs** | Minor | Simplified PUS limitation; not ST[19]-specific | Inherent to test_yamcs architecture. |
| 5 | **Event detection coupling with ST[05]** | Medium | ST[19] must observe ST[05] events; in simplified Python simulator, this requires an internal event bus | Add an `EventBus` singleton in the simulator; ST[05] simulator posts events; ST[19] handler subscribes. Or: ST[19] monitors incoming TM for ST[05] packets. |
| 6 | **TC[19,3] delete-all not in scope** | Info | Delete-all is a zero-payload TC; easy to add if needed later | Add `TC_19_3` MetaCommand with no arguments. |

### Gap 1 detail — Embedded TC packet

The PUS spec allows the "request" in TC[19,1] to be any valid TC. XTCE `MetaCommand`
elements cannot contain another `MetaCommand` as a payload field. There is no standard XTCE
way to express "variable-length opaque bytes" as a TC argument.

**Recommended simplified encoding:**
```
[fixed fields: N, app_pid, event_id] [request_tc_len: uint16] [request_tc_bytes: uint8×N]
```

**Alternative** (if all actions are the same TC type, e.g., always TC[17,1]):
Fix the request to 8 bytes; no length prefix needed. Simulator stores the 8 literal bytes.

### Gap 5 detail — Event detection

In the real YAMCS/Java environment (e.g., `Pus5Service.java`), ST[05] calls back to registered
handlers. In the Python simulator:

```python
class EventBus:
    def __init__(self): self._listeners = []
    def subscribe(self, fn): self._listeners.append(fn)
    def post(self, app_pid: int, event_id: int):
        for fn in self._listeners: fn(app_pid, event_id)

EVENT_BUS = EventBus()

# In ST[05] handler:
EVENT_BUS.post(app_pid=0, event_id=event_id)

# In ST[19] handler:
EVENT_BUS.subscribe(lambda apid, eid: on_event(apid, eid))

def on_event(app_pid: int, event_id: int):
    if not ea_function_enabled: return
    key = (app_pid, event_id)
    defn = ea_definitions.get(key)
    if defn and defn['status'] == 'enabled':
        tm_sock.sendto(defn['request'], (TC_DISPATCH_HOST, TC_DISPATCH_PORT))
```

#!/usr/bin/env python3
"""
PUS ST[19] Event-Action Service Simulator

Implements:
  TC[19,1]  - Add event-action definitions
  TC[19,2]  - Delete event-action definitions
  TC[19,4]  - Enable event-action definitions (specific or all)
  TC[19,5]  - Disable event-action definitions (specific or all)
  TC[19,6]  - Report status → sends TM[19,7]
  TC[19,8]  - Enable event-action function
  TC[19,9]  - Disable event-action function
  TC[19,10] - Report definitions → sends TM[19,11]
  TM[19,7]  - Event-action status report
  TM[19,11] - Event-action definition report

Non-standard extension: TC[19,1] request field uses a uint16 length prefix
before the raw TC bytes, since XTCE cannot encode variable-length TC packets.
"""

import logging
import signal
import socket
import struct
import sys
import threading
import time
from typing import Dict, Tuple

logging.basicConfig(level=logging.INFO,
                    format="%(asctime)s %(levelname)-8s %(message)s")
log = logging.getLogger("pus19")

TM_HOST = "127.0.0.1"
TM_PORT = 10035   # YAMCS UdpTmDataLink port
TC_HOST = "0.0.0.0"
TC_PORT = 10049   # YAMCS UdpTcDataLink sends here

APID = 190        # 0xBE

_seq_lock  = threading.Lock()
_seq_count = 0

# ── Global state ──────────────────────────────────────────────────────────────
ea_function_enabled: bool = False
ea_definitions: Dict[Tuple[int, int], dict] = {}
# key = (app_process_id, event_definition_id)
# value = {'status': 'enabled'|'disabled', 'request': bytes}
_state_lock = threading.Lock()


# ── EventBus (for ST[05] coupling) ───────────────────────────────────────────
class EventBus:
    def __init__(self):
        self._listeners = []
    def subscribe(self, fn):
        self._listeners.append(fn)
    def post(self, app_pid: int, event_id: int):
        for fn in self._listeners:
            fn(app_pid, event_id)

EVENT_BUS = EventBus()


# ── Sequence counter ──────────────────────────────────────────────────────────
def _next_seq() -> int:
    global _seq_count
    with _seq_lock:
        s = _seq_count
        _seq_count = (_seq_count + 1) & 0x3FFF
    return s


# ── TM packet builder ─────────────────────────────────────────────────────────
def _send_tm(tm_sock: socket.socket, subtype: int, data: bytes) -> None:
    secondary = struct.pack(">BB", 19, subtype)
    payload = secondary + data
    pkt_data_len = len(payload) - 1
    word1 = (1 << 11) | APID    # TM type=0, sec_hdr=1
    word2 = (0b11 << 14) | _next_seq()
    pkt = struct.pack(">HHH", word1, word2, pkt_data_len) + payload
    tm_sock.sendto(pkt, (TM_HOST, TM_PORT))


# ── TM[19,7] builder ──────────────────────────────────────────────────────────
def send_tm_19_7(tm_sock: socket.socket) -> None:
    with _state_lock:
        items = list(ea_definitions.items())
    n = len(items)
    body = struct.pack(">B", n)
    for (app_pid, event_id), defn in items:
        status = 1 if defn['status'] == 'enabled' else 0
        body += struct.pack(">HHB", app_pid, event_id, status)
    _send_tm(tm_sock, 7, body)
    log.info("TM[19,7] sent: %d entries", n)


# ── TM[19,11] builder ─────────────────────────────────────────────────────────
def send_tm_19_11(tm_sock: socket.socket,
                  keys=None) -> None:
    with _state_lock:
        if keys is None:
            items = list(ea_definitions.items())
        else:
            items = [(k, ea_definitions[k]) for k in keys if k in ea_definitions]
    n = len(items)
    body = struct.pack(">B", n)
    for (app_pid, event_id), defn in items:
        status = 1 if defn['status'] == 'enabled' else 0
        req = defn['request']
        body += struct.pack(">HHB", app_pid, event_id, status)
        body += struct.pack(">H", len(req)) + req
    _send_tm(tm_sock, 11, body)
    log.info("TM[19,11] sent: %d entries", n)


# ── TC handlers ───────────────────────────────────────────────────────────────
def handle_tc(data: bytes, addr, tm_sock: socket.socket) -> None:
    if len(data) < 8:
        return
    svc, sub = data[6], data[7]
    if svc != 19:
        return
    log.info("TC[19,%d] from %s", sub, addr)

    if sub == 1:   # Add
        _handle_add(data)
    elif sub == 2: # Delete
        _handle_delete(data)
    elif sub == 4: # Enable
        _handle_enable(data)
    elif sub == 5: # Disable
        _handle_disable(data)
    elif sub == 6: # Report status → TM[19,7]
        send_tm_19_7(tm_sock)
    elif sub == 8: # Enable function
        with _state_lock: global ea_function_enabled; ea_function_enabled = True
        log.info("Event-action function ENABLED")
    elif sub == 9: # Disable function
        with _state_lock: global ea_function_enabled; ea_function_enabled = False
        log.info("Event-action function DISABLED")
    elif sub == 10: # Report definitions → TM[19,11]
        keys = _parse_event_id_list(data)
        send_tm_19_11(tm_sock, keys if keys else None)
    else:
        log.warning("TC[19,%d] not handled", sub)


def _parse_event_id_list(data: bytes):
    """Parse N + N×{app_pid:uint16, event_id:uint16} from byte 8 onward."""
    if len(data) <= 8:
        return []
    offset = 8
    n = data[offset]; offset += 1
    if n == 0:
        return []
    keys = []
    for _ in range(n):
        app_pid = struct.unpack_from(">H", data, offset)[0]; offset += 2
        event_id = struct.unpack_from(">H", data, offset)[0]; offset += 2
        keys.append((app_pid, event_id))
    return keys


def _handle_add(data: bytes) -> None:
    offset = 8
    n = data[offset]; offset += 1
    with _state_lock:
        for _ in range(n):
            app_pid = struct.unpack_from(">H", data, offset)[0]; offset += 2
            event_id = struct.unpack_from(">H", data, offset)[0]; offset += 2
            req_len = struct.unpack_from(">H", data, offset)[0]; offset += 2
            req_bytes = bytes(data[offset:offset + req_len]); offset += req_len
            key = (app_pid, event_id)
            if key in ea_definitions:
                ea_definitions[key]['request'] = req_bytes
                log.info("  Updated EA def (%d,%d)", app_pid, event_id)
            else:
                ea_definitions[key] = {'status': 'disabled', 'request': req_bytes}
                log.info("  Added EA def (%d,%d) status=disabled", app_pid, event_id)


def _handle_delete(data: bytes) -> None:
    keys = _parse_event_id_list(data)
    with _state_lock:
        for k in keys:
            if k in ea_definitions:
                if ea_definitions[k]['status'] == 'enabled':
                    log.warning("  Cannot delete enabled EA def %s — rejected", k)
                    continue
                del ea_definitions[k]
                log.info("  Deleted EA def %s", k)


def _handle_enable(data: bytes) -> None:
    if len(data) == 8:
        with _state_lock:
            for defn in ea_definitions.values(): defn['status'] = 'enabled'
        log.info("  All EA definitions enabled")
    else:
        keys = _parse_event_id_list(data)
        with _state_lock:
            for k in keys:
                if k in ea_definitions:
                    ea_definitions[k]['status'] = 'enabled'
                    log.info("  Enabled EA def %s", k)


def _handle_disable(data: bytes) -> None:
    if len(data) == 8:
        with _state_lock:
            for defn in ea_definitions.values(): defn['status'] = 'disabled'
        log.info("  All EA definitions disabled")
    else:
        keys = _parse_event_id_list(data)
        with _state_lock:
            for k in keys:
                if k in ea_definitions:
                    ea_definitions[k]['status'] = 'disabled'
                    log.info("  Disabled EA def %s", k)


# ── Event-action trigger ──────────────────────────────────────────────────────
def on_event(app_pid: int, event_id: int, action_sock: socket.socket) -> None:
    """Called when an ST[05] event fires. Trigger matching enabled EA definition."""
    with _state_lock:
        if not ea_function_enabled:
            return
        defn = ea_definitions.get((app_pid, event_id))
        if defn is None or defn['status'] != 'enabled':
            return
        req = defn['request']
    log.info("EA trigger: event (%d,%d) → releasing %d-byte request", app_pid, event_id, len(req))
    # Send the stored request TC to the local TC dispatch port (or back to YAMCS via TC)
    action_sock.sendto(req, (TM_HOST, TC_PORT))


# ── TC listener ───────────────────────────────────────────────────────────────
def tc_listener(tm_sock: socket.socket) -> None:
    sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
    sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    sock.bind((TC_HOST, TC_PORT))
    sock.settimeout(1.0)
    log.info("TC listener on %s:%d", TC_HOST, TC_PORT)
    while True:
        try:
            data, addr = sock.recvfrom(4096)
            handle_tc(data, addr, tm_sock)
        except socket.timeout:
            continue
        except Exception as exc:
            log.error("TC listener error: %s", exc)


# ── Entry point ───────────────────────────────────────────────────────────────
def main() -> None:
    log.info("=== PUS ST[19] Event-Action Simulator ===")
    log.info("APID=%d  TM→%s:%d  TC←%s:%d", APID, TM_HOST, TM_PORT, TC_HOST, TC_PORT)

    tm_sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

    EVENT_BUS.subscribe(lambda ap, ei: on_event(ap, ei, tm_sock))

    threading.Thread(target=tc_listener, args=(tm_sock,), daemon=True).start()

    def _shutdown(sig, frame):
        log.info("Shutting down"); sys.exit(0)
    signal.signal(signal.SIGINT,  _shutdown)
    signal.signal(signal.SIGTERM, _shutdown)
    while True:
        time.sleep(1)


if __name__ == "__main__":
    main()
